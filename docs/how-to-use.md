# How You May Use the AI Operational Governance Framework (AI-OGF)

This page explains how individuals and organizations may use the AI Operational Governance Framework (AI-OGF) in a clear and practical way. It summarizes the key points of the AI‑OGF Limited Use License and the Permission and Collaboration policy.

For full legal terms, see:
- LICENSE.md
- permission.md

---

## 1. What You May Do Without Permission

You may use AI-OGF freely for the following purposes:

- Reading and viewing the framework on the official site
- Referencing the framework in research, analysis, or discussion
- Using the framework as inspiration to build your own internal governance processes
- Sharing links to the official AI-OGF site
- Forking or cloning the public repository for review or to propose contributions

These uses are allowed as long as you do not redistribute, modify, commercialize, or publish the framework.

---

## 2. What Requires Written Permission

You must request permission before using AI-OGF for any of the following:

### Commercial Use
- Consulting engagements
- Paid training or workshops
- Inclusion in commercial products or services
- Use in corporate governance offerings or toolkits

### AI, Automation, or Tool Integration
- Training AI models or LLMs on the framework
- Embedding the framework into software, tools, or automated systems
- Using the framework as source instructions for agents or orchestration platforms

### Redistribution or Publication
- Republishing the framework in any form
- Hosting it on another website or repository
- Creating derivative works or modified versions intended for distribution

### External Framework Development
- Building a competing or derivative governance framework
- Using AI-OGF as a base for a public or commercial standard

If you are unsure whether your intended use requires permission, contact the framework owner for clarification.

---

## 3. How to Request Permission

To request permission, provide:

- Your name
- Your organization (if applicable)
- A description of the intended use
- Whether the use is internal, external, or commercial
- Any timelines or relevant context

Send requests to:

randy.manthey (at) mantheyco.com

## 4. How to Request Collaboration Access

The repository is publicly visible for transparency.  
Write access is restricted to protect the integrity of the framework.

To request collaboration access, provide:

- Your name
- Your background or relevant expertise
- The areas of the framework you wish to contribute to
- Any affiliations or organizational context
- A statement confirming agreement with the Contributor Terms of Use

All contributions become the intellectual property of the framework owner.
