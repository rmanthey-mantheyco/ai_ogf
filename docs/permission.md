# Request Permission or Collaboration Access

The AI Operational Governance Framework (AI‑OGF) is made publicly available for **read‑only access** and **non‑commercial internal use**. This page explains how individuals and organizations may request permission for additional uses or apply to collaborate on the framework.

---

## Purpose of This Page

AI‑OGF is a proprietary framework developed and maintained by Randy Manthey.  
While the framework is publicly viewable, its source files, structure, and intellectual property are protected under the **AI‑OGF Limited Use License**.

This page provides a clear process for:

- Requesting permission for uses beyond internal reference  
- Requesting collaboration access to the private source repository  
- Understanding what is and is not allowed under the license  

---

# 1. Allowed Use (No Permission Required)

You may use AI‑OGF **without requesting permission** for:

- Reading and viewing the framework on the public GitHub Pages site  
- Referencing the framework in research, discussion, or commentary  
- Using the framework as inspiration to build **your own internal governance processes**  
- Sharing links to the official AI‑OGF site  

These uses are permitted as long as you **do not** redistribute, modify, or commercialize the framework.

---

# 2. Uses That Require Written Permission

You must request and receive **written permission** before:

### **Commercial Use**
- Consulting engagements  
- Paid training or workshops  
- Inclusion in commercial products or services  
- Use in corporate governance offerings or toolkits  

### **AI, Automation, or Tool Integration**
- Training AI models or LLMs on the framework  
- Embedding the framework into software, tools, or automated systems  
- Using the framework as source instructions for agents or orchestration platforms  

### **Redistribution or Publication**
- Republishing the framework in any form  
- Hosting it on another website or repository  
- Creating derivative works or modified versions intended for distribution  

### **External Framework Development**
- Building a competing or derivative governance framework  
- Using AI‑OGF as a base for a public or commercial standard  

If you are unsure whether your intended use requires permission, please reach out for clarification.

---

# 3. Requesting Permission

To request permission for any restricted use, please provide:

- Your name  
- Your organization (if applicable)  
- A description of the intended use  
- Whether the use is internal, external, or commercial  
- Any timelines or context that may be relevant

# 4. Requesting Collaboration Access

The AI‑OGF source repository is private to protect the integrity and intellectual property of the framework.

Individuals interested in contributing may request access by providing:

- Your name  
- Your background or relevant expertise  
- The areas of the framework you wish to contribute to  
- Any affiliations or organizational context  
- A statement confirming you agree to the Contributor Terms of Use  

Collaboration access is granted at the sole discretion of the framework owner.

All contributions become the intellectual property of the copyright holder.
